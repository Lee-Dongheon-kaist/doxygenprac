Detailed Description

: Meso class represents a single cell in the lane which is computed using meso-based model

Process GetFollowerLaneConnectionInfo: ComputeLaneChange -> MoveLaneChangeVehicles (from SpinLockVehicleQueue to corresponding lane change queue) -> MoveLaneChangeWaitingVehicles (Move vehicle from LaneChangeWaitQueue to SpinLockVehicleQueue by compare and insertion algorithm) position) -> ComputeICF -> MoveFromPreviousUnit (Move vehicle from previous units to wait queue and update the state) -> MoveWaitingVehicleToMainQueue (moves vehicle in the wait queue to main queue) -> UpdateVehiclePosition(Update position of the vehicles) -> Record

◆ ComputeLaneChange()

void Captain::Node::MesoUnit::ComputeLaneChange ( const std::unique_ptr< MesoComputationModel > & computationModel)

omputes LC to lane change from current unit stores result to m_laneChangeIndexVector

Process :

  1. Get average speed from left and right unit ptr (speed is set to zero if no left or right unit exists)
  2. Get max priority index of the vehicle
  3. If lane with maximum priority is current lane, then perform OLC otherwise, perform MLC to corresponding lane
  4. For OLC, computes possibility of straight, left, right lane changes and uses multinomial distribution to compute lane to lane change
  5. Store the results for both MLC case and OLC case into m_laneChangeIndexVector for use in MoveLaneChangeVehicle
    Parameters
    computationModelcomputationModel to use for computing LC

Check if current Lane ID is identical to vehicle's CurrentLaneIndex
